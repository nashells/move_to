# Unit: 設定管理

## 概要
移動先フォルダーの設定ファイルを読み込み、設定情報を提供する機能。

## 関連ユーザーストーリー
- US-007: 設定ファイルで移動先フォルダーを管理する

## スコープ

### 含まれる機能
- 設定ファイル（config.json）の読み込み
- 設定データの解析・バリデーション
- 移動先リストの取得API提供
- 設定ファイルが存在しない場合のデフォルト動作

### 含まれない機能
- コンテキストメニューへの表示（Unit: コンテキストメニュー）
- 実際のファイル移動処理（Unit: ファイル移動）

## 技術仕様

### 設定ファイル形式
```json
{
  "destinations": [
    {
      "displayName": "Temp",
      "path": "C:\\Temp"
    }
  ]
}
```

### インターフェース
```
Configuration
├── LoadConfiguration(): Configuration
├── GetDestinations(): List<Destination>
└── Destination
    ├── DisplayName: string
    └── Path: string
```

### バリデーションルール
- destinations配列は最大10件まで
- displayNameは空文字不可
- pathは空文字不可
- 11件目以降は無視

## 受け入れ条件
- [ ] 設定ファイルからdisplayNameとpathのペアを読み込める
- [ ] 最大10件まで読み込める
- [ ] 10件を超える設定は無視される
- [ ] 設定ファイルが存在しない場合、空リストを返す
- [ ] 不正なJSON形式の場合、エラーをログ出力し空リストを返す

## テスト観点
| テストケース | 期待結果 |
|-------------|---------|
| 正常な設定ファイル（3件） | 3件のDestinationリストを返す |
| 設定ファイルに11件登録 | 10件のみ返す |
| 設定ファイルが存在しない | 空リストを返す |
| JSON形式が不正 | 空リストを返す |
| displayNameが空文字 | その項目をスキップ |
| pathが空文字 | その項目をスキップ |

## 依存関係
- 外部依存なし（このUnitは他Unitから参照される基盤）

## 成果物
- 設定読み込みモジュール
- 単体テストコード
